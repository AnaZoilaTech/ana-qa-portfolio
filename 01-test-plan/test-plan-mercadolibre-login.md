# Test Plan – Login & Search Flow Testing (MercadoLibre)

**Project Title:** Login & Search Flow Testing – MercadoLibre  
**Tester:** Ana Zoila Rodriguez  
**Testing Types:** Functional Testing, Smoke Testing, Regression Testing  
**Environment:** Google Chrome – Desktop  
**Application Under Test:** MercadoLibre Website  

---

## 1. Scope of Testing
**In Scope:**
- Login page
- User authentication
- Search bar functionality
- Product listing results

**Out of Scope:**
- Payments
- Shipping module
- Chat support
- Mobile applications

---

## 2. Objectives
- Verify that the login functionality works as expected.
- Validate that the search bar returns accurate product results.
- Ensure UI elements (buttons, images, texts) load correctly.
- Identify bugs related to login and search flows.

---

## 3. Testing Types Used
- Functional Testing  
- Smoke Testing  
- Regression Testing  
- Exploratory Testing  

---

## 4. Risks & Assumptions
**Risks:**
- Broken images  
- Incorrect error messages  
- Slow page loading  
- Invalid search results  

**Assumptions:**
- User has access to valid login credentials.
- Website is online and stable during the test period.

---

## 5. Deliverables
- Test Plan  
- 15 Test Cases  
- 6 Bug Reports  
- API Tests (Postman)  
- SQL Queries used for QA  

---

## 6. Tools Used
- Google Chrome  
- Jira (for Bug Reports)  
- GitHub  
- Postman  
- SQL  

---

## 7. Acceptance Criteria
- User can log in with correct credentials.
- Incorrect login shows appropriate error message.
- Search bar returns valid product results.
- Pagination and filters work correctly.
- Logout clears session successfully.

---

## 8. Test Environment
- Browser: Chrome v142+
- OS: Windows 10 / Windows 11
- Network: Stable WiFi

---

## 9. Test Execution Summary
This plan was executed as part of a personal QA portfolio project. All tests were performed manually, and results were documented through test cases and bug reports.
